SUMMARY:

The djBase.js object library is made to allow inline json to be used in a HTML document.
Inline json that this js file reads may follow the rules for whitespace and apostrophes from
the HTML language.

Who is this document meant for?

This document is meant for developers with knowledge of
json, javascript, html, xml, xhtml, dhtml,
and some server side knowledge would be good, too.

How to use this document?

This document is broken up into sections with examples. Just scroll down to look through the examples.
In the top left corner of the page is a button to help you learn inline JSON, and in the top right corner are
some links to other more comprehensive documentation sites.

Documentation does not explain basic web concepts or terminology unless
unusual syntax is used.


Example One: "Activating the data-json Attribute"

By using the update() method we have activated the data-json attribute
for all div tags.

JSON parsed via update() uses the toJSON() method. This JSON "reader/parser"
allows inline JSON to be formatted like HTML ( White-space and apostrophe rules ). It also
stamps the autoTime property ( The time the JSON was created using the getTime() Date() object method. ),
and enters in commas between pairs for you but will throw an error if you place a comma after the last pair
( As long as you separate them with white space ).

IMPORTANT:
1.) Remember that the html property is not required but highly recommended, so that
you can leverage using the getJsonByTagName() method.

2.) Remember that the update method assigns the id, name, and class property values to the
element that holds the JSON via the setAttribute() method.
( These properties are required. If you don't assign values to the properties you will run into problems later. )

Example Two: "Using the Object Constructor"

Using the constructElement() we converted the first item in the div array into a javascript object.
( div[0] should contain the contain the json for the first element that was created. )

So from now on we can just use the _div object to gain access to data regarding the first div element.

In the next example we're going to retrieve the box property value from the div[0] via the addBoxSet() method.

Example Three: "Adding a Box Set"

What the addBoxSet does is create an array of nodelists that make up a custom set "type".
A box set is stored as an object property ( box: function(){} ). You can add the box property via JSON or Javascript.

A box set can be added by using json, javascript, or both. In this example we use both.
The example used the _div object for the addBoxSet method with the parameter of the first item in the div array.

Example Six: "Using the getElement() Method"

The getElement() has two methods of accessing it.
The first method is the function method.
The second is the String prototype method.
In either method the accepted parameter is a specially formatted list.
The list is a comma seperated list, where each item is a vertical bar
separated set of value and command pairs and the last pair must have a
semi-colon or comma after it. The accepted commands are:

1. "#" ( pound symbol ): getElementById()
2. "." ( period symbol ): getElementsByClassName()
3. "_" ( underscore symbol ): getElementsByName()
4. "" ( blank ): getElementsByTagName()

Example: "box1|#,box2|#,box3|#,box4|#,box5|#;"
( "command value"|"command" )
